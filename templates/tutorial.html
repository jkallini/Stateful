Finite State Machine Editor Tutorial

This page is an in-depth tutorial about how to use the Finite State Machine Editor, which you use to answer DFA practice problems and NFA practice problems. Each section below describes a different functionality of the FSM Editor, along with short demonstrations.

What is the FSM Editor?

The FSM Editor is the big white box displayed on every problem page. Each problem asks you to design a DFA or an NFA. In order to submit a solution, you must use the FSM Editor in order to draw the state diagram of your solution machine.

The sections below highlight the key functionalities of the Editor in detail, so that you can get started on some practice problems! If you would rather view a more concise description of how to use the Editor, you may refer to the Info button at the bottom of each problem page.

Adding a State

To add a state, double-click anywhere on the editor. You will see a green state appear, with a blinking cursor in its center. Make sure to give your state a label by typing while the cursor is blinking. If you want to submit a valid machine, each state should have a unique label.

You can move the state by clicking and dragging it around the Editor.

Adding a Transition

To add a transition between two distinct states, hold down shift, and click and drag your pointer from one state to the other. This will create a green transition arrow from the first state to the second state, with a blinking cursor next to it. Make sure to give your transition a label by typing while the cursor is blinking.

You can adjust the position of the transition by clicking and dragging it.

Adding a Self-Transition

To add a transition from a state back to the same state, hold down shift, and click on the state that you want to add the self-transition to. This will create a green transition arrow from the state back to itself. You can also add a label to a self-transition, and reposition it by clicking and dragging on the Editor.

Tip: Shorthand for Parallel Transitions

Sometimes, you may have two transitions with different labels that have the same start and end states. A short-hand way of representing these 'parallel transitions' is by using commas to separate the distinct input symbols that may cause the machine to transition from the current state to the next state. Deleting States and Transitions

To delete a state or transition, click on it and press the delete key (NOT the backspace key!). On Mac, this this is fn + delete. If you delete a state, all transitions to or from it will also be removed.

Adding a Start State

To indicate that some state is a start state, hold down shift, and click and drag your pointer from nowhere toward the state. This will create a green arrow pointing toward the start state. You may give it a label, like 'start' if you wish, but you may also leave it blank. By clicking and dragging, you can reposition the arrow.

Adding an Accept State

To make an existing state an accept state, double-click on the state. You may change it back to a non-accept state by double-clicking it again.

Typing Numeric Subscripts and Greek Letters

You may label states or transitions with numeric subscripts or Greek Letters.

To give a label a numeric subscript, type an underscore before the number. The range of possible subcripts is \(0\) to \(9\), inclusive. For example, to label a state as \(q_0\), type q_0.

To make a label a Greek letter, type a backslash before the name of the Greek letter. To type the capital form of the Greek letter, simply capitalize the first character of the letter's name. For example, to label a state as \(\Sigma\), type \Sigma, or to label a transition with \(\varepsilon\), type \epsilon.

Knowing how to type Greek letters is very important for drawing NFAs! NFAs may include epsilon transitions. The only way to write the \(\varepsilon\) character is by typing \epsilon. Otherwise, the Editor will think you are typing an invalid character.

Submitting an Answer

When you've drawn a DFA or an NFA and you're ready to submit your answer, click on the Submit button below the FSM Editor. A dialog box will appear. If you have not submitted a valid DFA or NFA, the dialog box will say 'Oops! Your FSM is invalid', and a brief message will describe your error. If your submission is a valid DFA or NFA, then the dialog box will either say 'Great Job!' or 'Incorrect' based on the correctness of your answer.

To clear the Editor and start over, you may press the clear Clear button. This cannot be undone, so use it with caution!
