Lesson 2: Notation and Formal Definitions


2.1 Formal Definition of a DFA

In the last lesson, we informally introduced the concepts of DFAs and NFAs, and we used state diagrams to represent them. While informal definitions are great for gaining an intuition about how finite state machines work, formal definitions are used most often in both computer science theory and applications. There are several reasons why having a formal definition is important, but here are a few of them:

  1. A formal definition is precise. If you were unsure about what is allowed or not allowed in a DFA/NFA, the formal definition will provide all of the information you need.
  2. A formal definition provides clear notation. You can easily express the machine you are trying to build if you use good notation.

What would you consider to be the defining features of a DFA? In this section, we will be formally defining DFAs and each of their features.

States

Let’s start with the DFA’s states. We formally define the set of states of a DFA using the symbol \( Q \). Generally in mathematics, a set is a collection of distinct objects, which we call the elements or members of the set. The elements of the set \( Q \) are the states of a given DFA. As an example, recall the DFA \( M_1 \) that we studied in the last lesson.

Figure 1.5: The deterministic finite automaton \( M_1 \).

Normally, sets are described formally by listing their elements in braces. So, we can write $$ Q_1 = \{ q_1, q_2, q_3\} $$ to denote the set \( Q_1 \) containing the states of the DFA \( M_1 \).

Alphabet

Another inherent feature of DFAs is the set of input symbols they take in. For instance, you probably noticed that \( M_1 \) only has \( 1 \)’s and \( 0 \)’s as labels on transitions. You couldn’t give it a string containing \( 2 \)’s or \( 3 \)’s, or English letters. That would be like giving a decimal integer or a string to a Java program that takes a binary integer as a command-line argument. The part of the DFA that enforces this restriction is its alphabet. The alphabet of \( M_1 \) only allows \( 1 \)’s and \( 0 \)’s as input symbols. We denote the alphabet of a DFA by the set \( \Sigma \), whose elements are the valid input symbols of the DFA. The alphabet of \( M_1 \) is \( \Sigma_1 = \{ 0, 1 \} \).

Transition Function

The transition function of a DFA, often denoted by the symbol \( \delta \), specifies exactly how the DFA moves from state to state. When we first looked at DFAs, there were two things we considered when determining how the machine would transition between states. First, we had to know the current state of the DFA. Let’s say this state is state \( x \). Second, we had to know the input symbol being read. Let’s say that symbol is \( 1 \). If the DFA’s state diagram had an arrow pointing from state \( x \) to state \( y \) labeled with a \( 1 \), we knew that the next state to go to was \( y \). We can think of the way a DFA transitions between states as a function whose first parameter is a state, or an element of \( Q \), and whose second parameter is an input symbol, or an element of \( \Sigma \). The output of the function is another state, or another element of \( Q \). We denote this using the following function notation: $$ \delta : Q \times \Sigma \rightarrow Q $$ This notation says that \( \delta \) is a function whose domain is \( Q \times \Sigma \), and whose range is \( Q \). Generally, \( A \times B \) is known as the Cartesian product of the sets \( A \) and \( B \), or the set of all ordered pairs where the first element is a member of \( A \) and the second element is a member of \( B \). In the example above, since \( x \) transitions to state \( y \) on input \( 1 \), we write \( \delta(x, 1) = y \). Let’s construct the transition function for \( M_1 \), which we will call \( \delta_1 \).

Below is the Cartesian product of \( Q_1 \) and \( \Sigma_1 \). $$ Q_1 \times \Sigma_1 = \{(q_1, 0), (q_1, 1), (q_2, 0), (q_2, 1), (q_3, 0), (q_3, 1)\} $$

The elements of the Cartesian product above specify the possible inputs to the transition function. We write out the function as follows: $$\delta_1(q_1, 0) = q_1$$ $$\delta_1(q_1, 1) = q_2$$ $$\delta_1(q_2, 0) = q_3$$ $$\delta_1(q_2, 1) = q_2$$ $$\delta_1(q_3, 0) = q_2$$ $$\delta_1(q_3, 1) = q_2$$ Functions that take ordered pairs as input are often expressed using two-dimensional tables. For the transition function of a DFA, each row corresponds to a state, and each column corresponds to an input symbol. Below is \( \delta_1 \) expressed as a table:
\( \delta_1 \) \( 0 \) \( 1 \)
\( q_1 \) \( q_1 \) \( q_2 \)
\( q_2 \) \( q_3 \) \( q_2 \)
\( q_3 \) \( q_2 \) \( q_2 \)
