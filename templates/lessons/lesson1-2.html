Lesson 1: Introduction to Finite Automata


1.2 Deterministic Finite Automata

In the last section, we gained a general intuition about a simple computational model known as a finite automaton. In this lesson, we will explore a specific category of finite state machine, known as the deterministic finite automaton (DFA). The following figure presents a state diagram for a DFA that we will call \( M_1 \). We will be using \( M_1 \) to gain an understanding of the different components that make up a DFA in this section.

Figure 1.5: The deterministic finite automaton \( M_1 \).

A DFA, unlike the general finite automata that we looked at in the last section, solves a pattern matching problem. We can think of the input to a DFA as a string of characters on a piece of tape. The DFA reads each character from the tape exactly once, moving from left to right. Once the DFA has read all of the characters and reaches the end of the string, it responds 'YES' if it accepts the string, and 'NO' if it does not accept the string. All strings that are accepted by the DFA are in the language of the DFA. The DFA is said to recognize the strings in its language.

To decide whether it will accept or reject a string, a DFA will observe the state it is in when it reaches the end of input string. If this state is an accept state, it will accept the string. If not, it will reject the string. We indicate accept states on diagrams using a double circle. In the machine \(M_1\) above, \(q_2\) is an accept state.

To help you understand how a DFA processes an input string to produce an output, step through the example below.


Figure 1.6: A visualization of the concept of a DFA. The transition diagram is part of the internal mechanism of a machine \(M_1\), and that machine reads strings character-by-character from an input tape.



The input string in this case is \(1101\). The machine's tape scanner starts at the first character, which is a \(1\).


The process begins in \(M_1\)'s start state, \(q_1\). The machine reads a \(1\) character from the tape.


After reading \(1\), \(M_1\) moves from state \(q_1\) to state \(q_2\), since this is the transition from the current state that has \(1\) as its label.


\(M_1\) is now in state \(q_2\). The machine reads a \(1\) character from the tape.


After reading \(1\), \(M_1\) moves from state \(q_2\) back to state \(q_2\), since this is the transition from the current state that has \(1\) as its label.


\(M_1\) is now in state \(q_2\). The machine reads a \(0\) character from the tape.


After reading \(0\), \(M_1\) moves from state \(q_2\) to state \(q_3\), since this is the transition from the current state that has \(0\) as its label.


\(M_1\) is now in state \(q_3\). The machine reads a \(1\) character from the tape.


After reading \(1\), \(M_1\) moves from state \(q_3\) to state \(q_2\), since this is the transition from the current state that has \(1\) as its label.


\(M_1\) is now in state \(q_2\). There are no more characters to read from the tape. Since \(q_2\) is an accept state, \(M_1\) ACCEPTS.
