Lesson 1: Deterministic Finite Automata


1.2 Defining DFAs Through Set Theory

In the last section, we gained a general intuition about a simple computational model known as a finite automaton. In this lesson, we will explore a specific type of finite state machine, known as the deterministic finite automaton (DFA). We will give a precise definition of a DFA in mathematical terms, and become more comfortable with common mathematical notation and definitions that are used in theoretical computer science. The following figure presents a state diagram for a DFA that we will call \( M_1 \). We will be using \( M_1 \) to gain an understanding of the different components that make up a DFA in this section.

Figure 1.5: The deterministic finite automaton \( M_1 \).

Some Set Theory

When describing DFAs, we refer to their components using precise mathematical notation. One of the most important mathematical concepts in computer science theory is the concept of a set. Generally in mathematics, a set is collection of distinct objects, which we call the elements or members of the set. Normally, sets are described formally by listing their elements in braces. So we can write $$ S_1 = \{ 5, 6, 2\} $$ to denote the set \( S_1 \) that contains the elements \( 5 \), \( 6 \), and \( 2 \). Since order does not matter when defining sets, \( \{ 2, 5, 6\} \) denotes the same set as \( S_1 \). The symbol \( \in \) denotes membership within a set. So, we can say that \( 5 \in S_1 \); this means that the number \( 5 \) is an element of the set \( S_1 \). Likewise, we can write \( 3 \notin S_1 \) to denote that \( 3 \) is not a member of the set \( S_1 \).

Another common way of defining sets is in what is called set-builder form. In set-builder form, a set is described by a property that each of its members must satisfy. for instance, writing $$ S_2 = \{x : x \text{ is a positive number less than 6} \}$$ has the same meaning as $$ S_2 = \{ 1, 2, 3, 4, 5\}$$ and reads as "\( S_2 \) is the set of all elements \( x \) such that \( x \) is a positive number less than 6." When using set-builder notation, we can also define members of a set in terms of another set: $$ S_3 = \{x \in S_2 : x > 3\}$$ This notation tells us that \( S_3 \) is the set of all elements in \( S_2 \) that are greater than \( 3 \). Considering what we know about \( S_2 \), we can see that \( S_3 = \{ 4, 5 \} \). Since every element of \( S_3 \) is also a member of \( S_2 \), \( S_3 \) is a subset of \( S_2 \), written as \( S_3 \subseteq S_2 \).

Every set is a subset of itself. For instance, say we have a set \(A\) and a set \( B \subseteq A \). It may be the case that \( B = A \). What if we want to specify that \( B \) is a subset of \( A \), but \( A \) and \( B \) are distinct? There is in fact a term for this. If \( B \) is a subset of \( A \), but \( A \) and \( B \) are distinct sets, then \( B \) is a proper subset of \( A \), written as \( B \subset A \).

Defining DFAs Formally

The sets \( S_1 \), \( S_2 \), and \( S_3 \) above are sets of numbers, but a set can have elements of any type, even other sets. To return to the topic of finite automata, we can formally define the attributes of a DFA using sets. One of the defining freatures of a DFA is its set of states, which we denote by \( Q \). For the machine \( M_1 \) above, its set of states is \( Q = \{ q_1, q_2, q_3 \}\). The start state, which we typically denote by \( q_0 \in Q \), is an element of a DFA's set of states. The start state of \( M_1 \) is \( q_2 \). The set of accept states of a DFA is typically denoted by \( F \), and since any accept state of a DFA is by definition in the set \( Q \), we know that \( F \subseteq Q \).
